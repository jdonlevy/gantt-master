Summary

Delivery Tracker provides a roadmap view that combines Jira Fix Versions → Epics → Stories with local-only UAT / Live / Milestone data. It includes multi-panel dashboards with rich-text updates, filters saved per dashboard, and a Gantt timeline view.

Key Features

Jira OAuth (3LO)

  • Sign in with Jira to fetch live roadmap data.
  • Session stored locally for API access.

Dashboards & Panels

  • Multiple rich-text panels per dashboard.
  • Fixed grid layout with width presets: Full / Two-thirds / Half / Third / Quarter.
  • Panel collapse / expand and delete (when signed in).

Rich Text Editor

  • Paragraph + headings
  • Bold / Italic / Underline / Strike / Code
  • Bullet, numbered, and checkbox lists
  • Links and images
  • Status/label pills

Filters (Saved Per Dashboard)

  • Projects (multi-select)
  • Statuses
  • Types (issue types)
  • Increment start / end

Gantt Timeline

  • Fix Version → Epic → Story nesting
  • Collapsible rows
  • Increment window clamps the visible range
  • UAT / Live markers
  • Milestone markers

Local‑Only Data

  • UAT date range per Fix Version
  • Live date range per Fix Version
  • Milestones (label + date + color)

How to Use

1) Sign In

Use the “Sign in to Jira” button in the header.

2) Open a Dashboard

Visit /dashboards/:slug (e.g., /dashboards/gpo).

3) Configure Filters

  • Select projects, statuses, and issue types.
  • Set increment start/end.
  • Save filters for the dashboard.

4) Update Panels

  • Edit panel content via the toolbar.
  • Add labels, links, lists, and images.
  • Save changes with the Save button.

5) Review the Timeline

  • Fix Versions are the top-level rows.
  • Expand to see epics and stories.
  • Update UAT/Live dates if needed (stored locally).

Notes / Limitations

  • Local-only fields do not sync back to Jira.
  • Panel layout is fixed grid (no drag/resize yet).
  • Access requires Jira authentication.
